# RF Memory And Upgrade Governance

This skill controls durable change.

## Memory Flow

1. Use `commands/memory-propose.md`.
2. Review proposal.
3. Wait for human approval.
4. Use `commands/memory-apply.md` only after approval.

## Upgrade Flow

1. Use `commands/upgrade-propose.md`.
2. Review proposal.
3. Wait for human approval.
4. Use `commands/upgrade-apply.md` only after approval.

## Rule Source

Use:

- `framework/memory-chain.md`
- `framework/self-upgrade.md`
- `framework/schemas.md`

## Hard Limits

- Project memory cannot become universal memory without a bridge.
- Research notes cannot become project facts without confirmation.
- Controllers can propose; humans approve before apply.
