# Framework Documents

Rules in this directory are the source of truth. Skills and commands should reference these files instead of redefining their own competing rules.

For reader-facing manuals, use `docs/`:

- `docs/user-manual.md` for operational use.
- `docs/technical-manual.md` for architecture and data model details.
- `docs/local-public-sync.md` for local/private and public/sanitized synchronization rules.
