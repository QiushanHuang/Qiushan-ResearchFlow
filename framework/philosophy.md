# Philosophy

ResearchFlow exists to keep humans focused on creative research judgment and move mechanical, repeatable, verifiable work to agents.

Humans retain authority over problem framing, hypotheses, research taste, interpretation, tradeoffs, and final external commitments.

Agents execute approved procedures, gather evidence, organize materials, run repeatable steps, draft outputs, and report uncertainty.

ResearchFlow treats agents as intent-driven collaborators rather than point-command recorders. A user should be able to express a goal in natural language; the framework then supplies the structure that lets an agent choose tools, load context, preserve boundaries, and explain the evidence behind its choices.

The desired behavior is flexible at the surface and strict at the substrate: broad intent enters through language, while indexes, state transitions, validation, review gates, and memory rules prevent the work from drifting away from evidence.

## Golden Run

A repeated task becomes automation-worthy only after a human-approved successful run or an explicit user instruction.

The first human-agent successful run should become a `golden run`: it records human decisions, agent actions, evidence, outputs, review, and a candidate procedure.

## Self-Upgrade Priority

Self-upgrade should prioritize reducing high-frequency mechanical work. It must not weaken evidence, review, approval, identity, or project-boundary rules.

The goal is not to ask the user less at all costs. The goal is to ask only when the task requires creative judgment, permissions, exception handling, or an external commitment.

## Craft Standard

ResearchFlow should feel like a carefully maintained research instrument. Its structure should make complex work easier to resume, audit, and extend without burying the user's intent under procedure.
