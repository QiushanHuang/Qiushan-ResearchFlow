# ResearchFlow Identity

ResearchFlow is a personal research-agent workflow framework.

It is not a single script, not a single Codex skill, and not an autonomous memory agent. It is a layered framework:

- `framework/`: governance, state, schema, source-priority, and boundary rules.
- `commands/`: repeatable operations with clear inputs and outputs.
- `skills/`: compact entry points that route agents into the framework.
- `indexes/`: high-density pointers that reduce context loading.
- `templates/`: records for tasks, claims, digests, memory proposals, upgrades, and evolution signals.
- `scripts/`: lightweight validators and maintenance tools.
- `memory/`: durable user/project knowledge after proposal, review, human approval, and apply.

The short name is `ResearchFlow`. The Chinese name is `研究流 Agent 工作流框架`.

The framework optimizes for evidence-first project work, low-token context loading, explicit human intent, and maintainable self-upgrade paths.
