# Command: Intake

Use intake to capture the task before deciding how much workflow is needed.

## Do

- Record the user goal in one sentence.
- Record expected output.
- Record constraints, deadlines, and explicit preferences.
- Identify whether the task touches files, experiments, research, memory, or framework rules.
- Move to `triage`.

## Output

```yaml
state: intake
goal:
expected_output:
constraints:
touches:
  files: false
  experiments: false
  research_library: false
  memory: false
  framework: false
unknowns:
```
