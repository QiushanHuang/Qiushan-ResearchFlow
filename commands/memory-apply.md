# Command: Memory Apply

Use memory-apply only after explicit human approval.

## Do

- Verify the approved proposal id.
- Write to the correct memory layer.
- Preserve source and scope.
- Record apply date and actor.
- Record how to revise or supersede the memory later.

## Forbidden

- Applying memory from controller judgment alone.
- Writing project-specific parameters into universal memory.
- Writing research-library notes as project fact without confirmation.

## Output

```yaml
apply_id:
proposal_id:
target_path:
applied_by:
approved_by:
applied_at:
supersedes:
revision_policy:
```
