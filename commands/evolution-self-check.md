# Command: Evolution Self-Check

Use this when the user asks ResearchFlow to summarize problems, self-upgrade opportunities, stale candidates, unresolved bugs, or repeated habits.

## Check

- open inbox signals;
- triaged items without candidates;
- candidates without proposal or decision;
- stale project links;
- resource records with old `last_seen_at`;
- validator failures and warnings;
- repeated user preferences that may need memory proposal.

## Output

Write a self-check record. Propose next actions, but do not apply memory or upgrades.
